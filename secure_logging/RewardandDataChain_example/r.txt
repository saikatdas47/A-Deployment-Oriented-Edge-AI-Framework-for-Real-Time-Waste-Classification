	
    Implementation of a Private Blockchain for IoT Waste Management

1. Introduction

The Internet of Things (IoT) generates massive amounts of data, including waste disposal activity in smart cities. Ensuring the integrity, authenticity, and traceability of this data is critical. Blockchain technology provides a decentralized, tamper-proof ledger that can be applied to track waste disposal events and user activity.

In this project, a private blockchain was implemented to log user and waste data with the following goals:
	•	Prevent duplicate entries.
	•	Ensure data integrity using cryptographic hashes.
	•	Provide authentication using digital signatures.
	•	Enable easy verification of the entire data chain.

⸻

2. System Architecture

The system consists of three main components:
	1.	Data Sources
	•	userinfo.csv contains user registration data:
	•	VoterID, Name, FingerID, Timestamp
	•	wastelist.csv contains waste disposal events:
	•	DateTime, BinID, VoterID, Name, ClassifiedWaste
	2.	Blockchain Storage
	•	Blocks are stored locally as JSON files.
	•	Blockchain folder: /Users/saikatdas/Desktop/IOT Project/New/MIne/BlockChain/Blockchain/
	3.	Cryptographic Security
	•	Each block contains:
	•	Index
	•	Timestamp
	•	Data (from CSV)
	•	Previous block hash
	•	Block hash
	•	Digital signature of the block hash
	•	Hash Algorithm: SHA256
	•	Digital Signature: RSA (2048-bit) with PSS padding

⸻

3. Blockchain Implementation

3.1 Block Structure

Each block contains the following:
    
    
    {
  "index": 0,
  "timestamp": "2025-09-11 21:29:16",
  "data": {...},
  "prev_hash": "0",
  "hash": "abcd1234...",
  "signature": "deadbeef..."
}

    
    •	prev_hash ensures blocks are linked sequentially.
	•	hash ensures data integrity.
	•	signature ensures authenticity.

3.2 Duplicate Prevention
	•	Each block is checked against existing blocks based on a unique key:
	•	Timestamp for user info
	•	DateTime for waste events
	•	If the key already exists, the block is skipped, preventing duplicate entries.

3.3 Digital Signatures
	•	Signing: Private key signs the hash of the block.
	•	Verification: Public key verifies the signature to ensure authenticity.
	•	This prevents unauthorized modifications.

3.4 CSV to Blockchain Process
	1.	Load blockchain JSON file (or create a new chain if it does not exist)
	2.	Read CSV file line by line
	3.	For each row:
	•	Check for duplicates
	•	Add block with:
	•	Hash
	•	Signature
	•	Link to previous block
	4.	Save updated blockchain

3.5 Blockchain Verification
	•	Each block is checked for:
	•	Hash integrity
	•	Signature validity
	•	Previous hash linkage
	•	If all checks pass, the blockchain is verified successfully.

5. Potential Improvements
	1.	Hash Algorithm: Replace SHA256 with SHA3-256 or BLAKE3 for higher performance and security.
	2.	Digital Signature Algorithm: Use Ed25519 for faster signing and verification.
	3.	Merkle Tree Structure: For efficient verification of individual transactions.
	4.	Consensus Protocols: Implement PoW or PoS for distributed blockchain systems.
	5.	Serialization: Use MessagePack or CBOR for compact storage instead of JSON.
